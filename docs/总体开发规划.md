## 一、项目总览

### 1.1 项目目标

将开源项目 `chatlog_alpha`（Go 语言编写的数据解析与 AI 语义检索引擎）封装为一款**开箱即用**的跨平台桌面应用，实现以下核心能力：

| 目标 | 描述 |
|------|------|
| **零配置启动** | 用户双击图标即可使用，无需手动启动终端、配置环境变量或管理端口 |
| **进程透明管理** | 自动启动/守护/清理 Go 后端 Sidecar，彻底消除僵尸进程与端口占用问题 |
| **Apple 风格高级 UI** | 毛玻璃材质、弹簧阻尼动画、Squircle 圆角、无边框窗口、系统字体自适应 |
| **聊天数据浏览** | 完整的聊天记录展示、搜索、筛选、统计功能 |
| **AI 语义增强** | 基于本地 Ollama 的语义检索与 RAG 流式问答 |
| **知识图谱** | 3D 可视化展示人物/事件/话题关联网络 |
| **隐私与安全** | 本地数据隔离、隐私模式、防遥测 |

### 1.2 技术边界

| 维度 | 决策 | 说明 |
|------|------|------|
| 后端核心 | 封装现有 chatlog_alpha Go 二进制 | 不改写后端逻辑，作为 Sidecar 挂载 |
| 通信协议 | HTTP REST + SSE | 前端通过本地 HTTP 与 Go 后端通信 |
| 前端框架 | React 18 + TypeScript 5 | 类型安全贯穿全链路 |
| 桌面框架 | Tauri v2 | 原生 Sidecar 支持，Rust 层进程管理 |
| 状态管理 | Zustand | 轻量、高性能，适合作数据员角色 |
| 动画引擎 | Framer Motion | 基于物理的弹簧动画 |
| 3D 渲染 | Three.js | 知识图谱可视化 |
| 目标平台 | macOS (x64 + arm64) + Windows (x64) | 双平台同步交付 |

### 1.3 开发策略

采用**核心先行、逐步迭代**策略：

```
Sprint 1 (基础) → Sprint 2 (聊天) → Sprint 3 (AI) → Sprint 4 (图谱) → Sprint 5 (高级) → Sprint 6 (发布)
     ↓                  ↓                ↓               ↓               ↓               ↓
  可运行壳         MVP版本          语义增强版      可视化增强版     高级体验版      正式发布版
```

每个 Sprint 结束时产出可验收的功能增量。总体规划完成后，每个 Sprint 启动前单独制定详细的模块级开发规划。

## 二、架构总设计

### 2.1 Mediator 四层架构全景

```
┌─────────────────────────────────────────────────────────┐
│                    L1 入口层 (Entry)                      │
│  页面路由 · 空间布局 · 事件委托                            │
│  DashboardView / ChatView / GraphView / SettingsView     │
├─────────────────────────────────────────────────────────┤
│                  L2 协调层 (Coordinator)                   │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌────────────┐  │
│  │ 指挥官    │ │ 数据员    │ │ 外交官    │ │ API文档     │  │
│  │ Commander │ │ DataClerk │ │ Diplomat  │ │ APIDocs    │  │
│  └──────────┘ └──────────┘ └──────────┘ └────────────┘  │
│  业务编排      Zustand状态   网络策略      TypeScript类型  │
├─────────────────────────────────────────────────────────┤
│                  L3 分子层 (Molecule)                      │
│  MessageBubble · ContactList · SearchPanel · GraphNode   │
│  StatCard · QAPanel · SettingsForm · PrivacyToggle       │
├─────────────────────────────────────────────────────────┤
│                  L4 原子层 (Atom)                          │
│  ┌─────────────┐  ┌──────────────┐  ┌─────────────────┐  │
│  │  UI 原子     │  │  网络原子     │  │  系统原子        │  │
│  │ AppleButton  │  │ fetchHistory  │  │ spawnSidecar    │  │
│  │ GlassPanel   │  │ streamQA      │  │ killPort        │  │
│  │ Avatar       │  │ searchData    │  │ readOsTheme     │  │
│  │ Typography   │  │ fetchStats    │  │ watchWindow     │  │
│  └─────────────┘  └──────────────┘  └─────────────────┘  │
├─────────────────────────────────────────────────────────┤
│              Rust 原生层 (Tauri Core)                      │
│  端口猎杀 · 进程守护 · 窗口管理 · 系统API                    │
├─────────────────────────────────────────────────────────┤
│          Go Sidecar (chatlog_alpha 引擎)                   │
│  HTTP API · SSE Stream · 数据库解密 · AI推理               │
└─────────────────────────────────────────────────────────┘
```

### 2.2 核心约束（架构铁律）

| 约束 | 对象 | 说明 |
|------|------|------|
| **禁止相互调用** | L4 原子 ↔ L4 原子 | 原子间完全独立，通过上层组装 |
| **禁止相互调用** | L3 分子 ↔ L3 分子 | 分子间完全独立，通过 L2 协调 |
| **禁止业务状态** | L4 原子层 | 不含任何业务逻辑或全局状态读取 |
| **禁止直接调用网络** | L3 分子层 | 分子不发起网络请求，数据由上层注入 |
| **L1 不含逻辑** | L1 入口层 | 仅布局和事件委托，不含 if/else 分支 |
| **所有跨模块交互经 L2** | 全局 | L2 是唯一的协调中枢 |

### 2.3 数据流转示意

```
用户操作 → L1 (事件委托)
              ↓
         L2 Commander (业务编排)
              ↓
    ┌─────────┼─────────┐
    ↓         ↓         ↓
DataClerk  Diplomat  (调用L4)
(状态更新)  (网络策略)
    ↓         ↓
  Zustand   L4 Network Atoms
  Store     (HTTP/SSE请求)
    ↓         ↓
L3/L1       Go Sidecar
重新渲染     返回数据
```

| 任务 | 内容 | 关键文件 |
|------|------|---------|

| **端口猎杀** | 启动前探查并清理占用端口的僵尸进程 (macOS: lsof, Windows: netstat) | src-tauri/src/main.rs |

| **优雅退出** | 拦截窗口关闭事件，发送终止信号，等待子进程退出后再结束主程序 | src-tauri/src/shutdown.rs |

### Sprint 3: AI 聊天分析 (P2 — 可选增强)

> **说明**: 本模块为可选增强功能，核心价值在于让用户能够用自然语言提问，让 AI 基于其历史聊天记录进行分析并给出回答。典型场景如："过去一年我和谁聊得最多？"、"帮我总结一下上周和老板的对话要点"。**Sprint 1-2 的软件构建是绝对主角，本 Sprint 为锦上添花。**

**目标**: 通过接入本地 Ollama 模型，利用已导出的聊天数据库作为知识源，实现基于历史对话内容的智能问答。

#### 3.1 聊天智能问答

| 分子组件 | 功能 | 关键交互 |
|---------|------|---------|
| `ChatQA` | AI 问答界面，用户自然语言提问，AI 基于聊天记录回答 | 打字机逐字渲染效果 |
| `QAMessage` | 单条问答消息（用户问题 + AI 回答 Markdown 渲染） | 自动滚动锚点 |
| `QAInput` | 问题输入框 | 回车提交 |

#### 3.2 分析功能

| 分子组件 | 功能 |
|---------|------|
| `ConversationAnalyzer` | 让 AI 对指定联系人或时间段的聊天进行总结、情绪分析、话题提取 |

#### 3.3 L2 协调层扩展

| 角色 | 新增职责 |
|------|---------|
| Commander | `askQuestion(query)`, `analyzeConversation(contactId, timeRange)` |
| Diplomat | SSE 流式数据解析与缓冲，503 过载错误友好翻译 |
| DataClerk | 问答历史存储、分析结果缓存 |

| **L3** | AppLayout, Sidebar, StatusBar | ContactList, ChatView, MessageBubble, MessageList, GlobalSearch, DashboardOverview, TrendChart | SemanticSearch, QAPanel, QAMessage, IndexStatus | GraphEngine, GraphNode, GraphEdge, GraphTooltip | PrivacyToggle, DevConsole, SettingsPage, AppearanceSettings | 无新增 |
| **L4-UI** | AppleButton, GlassPanel, Avatar, Typography, SkeletonLoader, SpringModal | 复用 | 复用 + CodeBlock, MarkdownRenderer | 复用 | 复用 + BlurOverlay | 复用 |
| **L4-Net** | fetchDbStatus | fetchHistory, fetchSearch, fetchStats, fetchContacts | streamQA, fetchSemanticTopics, rebuildIndex | fetchGraphVisualize, fetchGraphQuery | 复用 | fetchUpdate |
| **L4-Sys** | spawnSidecar, killPort, readOsTheme | detectWxPath, openDirectoryPicker | 复用 | 复用 | readVibrancy, watchThemeChange | 复用 |

### 5.2 禁止依赖检查清单

| 检查项 | 是否违反 | 处理方式 |
|--------|---------|---------|
| L4 原子 A 是否 import L4 原子 B | 禁止 | 原子间无 import，通过 L3 组装 |
| L3 分子 A 是否 import L3 分子 B | 禁止 | 分子间无 import，通过 L2 坐标注入数据 |
| L3 分子是否直接 import L4 网络原子 | 允许 | L3 通过 props 接收数据，不自己调用网络 |
| L2-Cmd 是否直接操作 DOM | 禁止 | L2 只操作状态和数据，由 L1/L3 响应渲染 |
| L1 页面是否包含 useState 业务状态 | 禁止 | L1 只做布局和事件委托 |

## 六、关键技术风险与对策

| 风险 | 影响 | 概率 | 对策 |
|------|------|------|------|
| Sidecar 端口被占用导致应用无法启动 | 阻断性 | 高 | Rust 层实现前置端口猎杀 (lsof/netstat + kill)，等待 500ms 确认释放 |
| Go 二进制交叉编译失败 | 阻断性 | 中 | 在 Sprint 1 即搭建 CI 交叉编译流水线，尽早验证 |
| 微信数据库版本升级导致解密失败 | 功能性 | 中 | L2 Diplomat 拦截原始报错，翻译为友好提示；预留更新机制 |
| Ollama 引擎未安装导致 AI 功能不可用 | 功能性 | 中 | 应用内检测 Ollama 可用性，未安装时显示引导，AI 功能为可选增强而非阻断 |
| macOS 公证被拒 | 阻断性 | 中 | 提前申请 Apple Developer 证书，在 Sprint 4 就开始试公证 |
| Three.js 在低配机器上性能不足 | 体验性 | 低 | 实现 LOD 降级策略，节点数过多时自动切换 2D 降维视图 |
| 大量聊天记录导致虚拟滚动性能问题 | 体验性 | 高 | 使用 react-window/react-virtuoso 虚拟滚动，消息分页加载 |

### 7.2 代码规范

| 规范项 | 要求 |
|--------|------|
| 类型安全 | 所有组件和函数必须显式定义 TypeScript 类型，禁止使用 `any` |
| 注释策略 | 仅在复杂逻辑处添加注释，简单代码自解释。不编写冗余注释。 |
| 组件拆分 | 单个文件不超过 200 行，组件职责单一 |
| 命名 | 组件：PascalCase；函数/变量：camelCase；常量：UPPER_SNAKE_CASE |
| 导入顺序 | React → 第三方库 → L2 → L3 → L4 → 样式 |
| 原子/分子隔离 | L3/L4 组件文件中禁止 import 同级其他组件 |
